Building Infrastructure-Grade JSON Canonicalization in Go

json-canon is a Go library for RFC 8785 JSON Canonicalization Scheme. It exists because deterministic serialization is a harder problem than it appears â€” and because the infrastructure that depends on it deserves more than the tests pass as a correctness argument.

These articles document what it took to build: the algorithms, the parser, the architectural decisions that separate a library from dependable infrastructure, and the evidence-based release process that proves the output is actually deterministic. Each article is self-contained, but together they trace the full arc from specification compliance to provable correctness.
